Introduction

 

Stock Controlling System is specially developed for managing stock of books of Singapore Informatics Computer Institute (Pvt) Ltd.

It helps maintaining an accurate stock of books all the time. All the books Issues and Receipts can be easily updated.

You can get Stock Related Reports in many ways. It includes a powerful System Maintenance System including Messaging System,

Database Backup/Restore System and has been specially developed for Multi-user Environment, which handles the database on the server.

Getting Started

 

1.       Install the Software by running the Setup.exe file.

2.       When you are prompted for Database Configuration, select the relevant option and configure the database.

 

The options describe in 4 different ways.

 

1.)    Database Locates on Local Computer.

2.)    Database Locates on a File Server mapped by a Network Drive.

3.)    Database is in a Shared Location on a File Server.

4.)    Convert an existing Database Backup File (*.dbf) as the Database.

 

 

3.       After the Database configuration is successfully done, you will be prompted to the User Login,

where you should enter the Initial Password as ‘password’ for the User Name, Administrator.

Once you log in change the Administrator password.

Data Entry Sub System

 

 This System includes 6 different Data Entry Interfaces which handles all the Main Data of the System.

These interfaces can be accessed from the Main Menu.

 

1.)    Master Data Entry Interface

2.)    Books Details Entry Interface

3.)    Current Stock Interface

4.)    Books Issue Entry Interface

5.)    Books Receipt Entry Interface

6.)    Books Not In Use Entry Interface

 

Back to top

1.) Master Data Entry Interface (Step 1)

 

These entries are for Master Files which will entirely handle the Main Data Flow of the system. It is a must to enter the information

for Master files to be usable the system. With this interface you should enter all the Courses, Modules related to the books in stock.

As Categories you can enter (IT, Business, Business/IT, PSD, Course Manuals ect.).

Since all the other main entries of the system depend on these entries, those must be accurate.

(No duplicate entries allowed.)

 

When a new book, which is of a new Course, Module is introduced, you should update the Master Information Entry first.

 

* Study the sample entries provided with Stock Controlling System.

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

Back to top

2.) Books Details Entry Interface (Step 2)

 

After entering Master Information of the system you must enter books Details entries, where you should match the particular Module

with the relevant Course and the Category. These entries are also very significant to the data flow of the system.

No duplicate entries allowed.

 

Note: If you amend Master Information you should amend the books details entries as well.

( Amendment includes Record Deletion, Adding, Editing.)



 

Back to top

3.) Current Stock Interface (Step 3)

 

This is the Stock Controlling System initial Current Stock Entry Interface, where you should enter the data for the first time.

 (No duplicate entries allowed.)

 

 

 

If you need, amendments can be done with this interface even later.

 

Back to top

4.) Books Receipt Entry Interface (Step 4)

 

With this interface you enter all the Books Receipts which adjusts the Current Stock.

When you receive a new book, which the Master Information and Books Details has been updated

The Current Stock Entry for that will be automatically added. This is a very receptive interface which supports

for all the record amendments including Record Deletion, Adding, Editing and is responsible for keeping the Current Stock Up-to-date.

 

 

Back to top

5.) Books Issue Entry Interface (Step 5)

 

This Interface handles all the Books Issues and also very sensitive for quantity

adjustments, record adding, record deleting and always keep the current stock Up-to-date.

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 


Back to top

6.) Books Not In Use Entry Interface (Step 6)

 

As Stock Controlling System handles all the books in the stock you can enter

the details of books which are not in use (Old Books) using this interface.

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

** The interfaces describes in Step 1, 2, 3 and 6 are updated rarely as most of the data entering with these interfaces

are entered as initial data of the Stock Controlling System.

 

** The Data of interfaces with Step 4, 5 are most updated.

The key features of Messaging System

 

* Capable of sending messages among users within the Stock Controlling System. (Messages can be sent either One User or All Active Users.)

 

* Reply to Sender option helps you to quickly feedback the sender.

                                   

* With Messaging System there are some steps which are specially allocated for Administrators.

 

1.)    Send User System Log Out Messages

 

This feature helps the Administrators to log out the user/users using the system at any moment.

This is useful to log out the active users before the Database Restoration is done, Database Location on the server is changed and ect.

 

2.)    Send User System Log Off Messages

 

Using this option you can log off the users at any moment.

             

 The Message Types of Stock Controlling System are categorized into 4 types.

 

Back to top

Message Types:

 

PRIVATE –

This type includes private messages which are sent by one user to another user.

(These messages are private and hidden from others.)

              

PUBLIC –

This includes Public messages which are sent by one user to all the other active users.

(Useful to Administrators to send critical messages to all active users of the system.)

 

USER LOG OFF NOTICE –

This type is used by Administrators to log off a particular user or all active users.

 

SYSTEM LOG OUT NOTICE –

Administrators can use this type to log out a specific user or all active users.

Backup

 

With Backup Database section you should select a backup path where the backup files are saved.

Backup files are saved with the date and time when it is backed up.

This is useful to easily pick the relevant backup file to restore.

 

Back to top

Restore

 

You can restore the backup files when it is needed using this option. Before start restoring make sure

that no other users have logged in the system.

You can monitor Active User Status with this interface to ensure that the other users have logged out the system.

 

* With Delete Backup File option you can delete the backup files created.

 

* By checking the value of ‘Create a Backup and Prompt me Automatically…’

 You can create a backup of the database when the Stock Controlling System loads and it will show the Backup/Restore Database interface.

 

Note: If you have not specified a backup saving location it will create the Database Backup File in the same location where the Database exists.

 

* By checking the value of ‘Prompt me when programme loads.’ Stock Controlling System will simply show the Backup/Restore Database interface when it loads.

9.) Clear Database Location

 

If the database location has been changed, you can use this option to clear the current database configuration.

 

This can be accessed from Main Menu  > Utilities >  Clear Database Location.

 

           

 

 

 

 

 

 

 

 

When you run the Stock Controlling System again after the Database Location is cleared, you will be prompted to the Database Configuration Interface,

where you should reconfigure the Database location by selection the relevant option.

 

 

 

 

 

 

 

 

 

 

 

 

 

                                                          

 

 

 

 

 

 

 

 

The fourth option, which is mentioned as Convert an existing Database Backup File (*.dbf) as the Database would be important when an occasion

where you can not access the database which is on the server due to network problems. By using this option you can convert the latest Database Backup File

as the Active Database and use that database to enter the data.

After the network problem is resolved you can backup the local database and clear the local database location as described above.

Then configure the database on the server again and do a database restoration with the previous backup you created.
